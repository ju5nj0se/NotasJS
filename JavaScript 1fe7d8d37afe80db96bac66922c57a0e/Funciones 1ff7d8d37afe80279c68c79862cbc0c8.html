Funciones

Son bloques de codigo reutilizables que nos sirven para organizar mas el codigo.

  • Se declaran mediante la palabra reservada function .
  • Para retornar valores utilizamos return .
  • Pueden recibir valores a los que se les llaman parametro.
  • Deben tener un nombre.
  • Puedo llamarla desde cualquier parte del codigo, incluso si fue declarada antes. A esto se le llama ‘hosting’

Ejemplo de una funcion clasica

function saludar(nombre) {

let saludo = "Bienvenido" + nombre;

return saludo;

}

saludar("juanjose");

Antes de comenzar con las otras funciones necesitamos entender que es un callback.

Funciones anonimas

  • No se les asigna un nombre en especifico.
  • No tiene hoisting.

Ejemplo

let saludar = function(nombre) {

return "Bienvenido" + nombre;

}

Funciones flecha

  • Son funciones anonimas.
  • No necesitan de la palabra reservada function .
  • No es necesario la palabra reservada return .
  • Si solo tengo un parametro no es necesario poner parentesis.
  • Si no tengo parametros es obligatorio poner los parentesis vacios.

Ejemplo

let saludar = nombre => "Saludos " + nombre;//Con 1 parametro

let saludar = (nombre, edad) => "Hola " + nombre + " tienes " + edad " años";//Con mas parametros

let saludar = () => "Hola Mundo";//Sin parametros

Callback

Se le nombra callback a una funcion que se pasa como parametro a otra funcion y que es ejecutada despues (esta claro que debe ser llamada).

Explicacion del codigo

  • Se define la funcion ‘ejemplo’ con 2 parametros, el segundo parametro es la palabra reservada callback que por medio de ella le estamos diciendo a JS que el segundo parametro va a hacer otra funcion. Por ultimo se llama a la funcion callback dentro del bloque de codigo de la funcion ‘ejemplo’.
  • Se declara una variable
  • Se llama a la funcion ‘ejemplo’ con el primer parametro que es la variable y el segundo que es una funcion, una funcion flecha. Esta funcion es anonima, por lo tanto no tiene nombre ademas de que se puede declarar en la misma invocacion y se le asigna un bloque de instrucciones en su declaracion.
