Manejo de errores en JS

throw

Con esta palabra reservada podemos generar errores especificos para situaciones particulares.

Ej.

try/catch

Este bloque permite manejar los erroes de manera adecuada durante la ejecucion del codigo. Si ocurre un ocurrer en el bloque try, el control pasa al bloque catch.

Ademas existe el finally que se ejecuta siempre, haya o no error.

Operador de encadenamiento opcional

Es una caracteristicas mdoerna que permite acceder a propiedades y metodos de objetos sin lanzar un error si un propiedad no existe (en lugar de lanzar un error).

Manejo de errores global

Podemos capturar errores globalmente utilizadno window.onerror para evitar que errores no capturados detengan la aplicacion.
